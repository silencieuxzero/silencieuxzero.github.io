概述
BMCN OS（全称后室主要探险者集团中文操作系统，Backrooms Major explorer group Chinese Operating System），是主要探险者集团中文部所使用的电子操作系统。该操作系统可在M.E.G.CN制作的x49_10架构的灵巧终端上运行，目前版本代号为：甲子（BMCN OS6.0）。
BMCN OS尽全力为用户提供最稳定的软件，以使流浪者可以在任何情况下正常使用BMCN OS来获取信息。初始安装完成的BMCN OS只是一个稳定的基本系统，随后用户可以根据自己的喜好安装需要的软件并配置成符合自己理想的个性化系统。

系统特点
高效的运行效率
BMCN OS的运行效率相当之高，一方面是因为我们摒弃了操作系统应有的美观，转而采用了高效和快速;另一方面是因为我们将选择权留给了用户，当用户的BMCN OS安装完成之后，他只会得到一个基本的操作系统和桌面环境。这对于在危险楼层中急于查阅信息的流浪者来说是关键的，因为过于冗杂的系统和花哨的桌面系统会大大提升系统的不稳定性，而在后室，这种不稳定性将会是致命的。因此我们一直将选择权留给用户，这不仅仅是对用户体验感的负责，也是对用户生命的负责。

丰富的桌面环境
BMCN OS是一个类unix操作系统，默认桌面环境为主要探险者集团中文分部桌面环境（Main Explorer Group Chinese Branch Desktop Environment），用户也可以在安装过程中在选择框中选择自己喜爱的一款桌面环境，目前BMCN OS所提供的桌面环境有以下几个：
主要探险者集团中文分部桌面环境（Main Explorer Group Chinese Branch Desktop Environment）——主要探险者集团中文分部研发
天空桌面环境（Sky Desktop Environment）——后室通信委员会研发
联邦桌面环境（Federated Desktop Environment）——后室合众国研发
Hirasaw Adai桌面环境（Hirasaw Adai Desktop Environment）——BMCN OS社区项目
由于部分组织/社区所开发的桌面环境界面的应用仍然处于不稳定状态，因此BMCN OS stable版本官方支持桌面环境仅包含以上四种，如您想安装其他的不受M.E.G.CN支持的桌面环境，请安装BMCN OS testing版本或unstable版本。
众多的版本分支
BMCN OS拥有诸多分支，这些分支往往是由其他组织或社区开发而成，因此不受官方支持，但一些足够优秀的分支会受到M.E.G.CN官方支持，以下是一些常见的分支：
BMCN OS Multilingual Edition（BMCN OS多语言版）——该分支由自由通信组织（Free Communications Organization）创建并开发，该分支旨在帮助其他语言分部的流浪者使用BMCN OS操作系统，目前该分支处于活跃状态且发展良好，最新版本号为1.3.7
BMCN Pro OS（BMCN OS专业版）——该分支由后室极客组织（Backroom Geek Organization）创建并开发，该分支旨在将BMCN OS变得更加开放以便于开发者使用，该分支在原有的基础上加入了大量专业功能以便于极客、开发者和计算机爱好者使用，目前该分支处于活跃状态且发展良好，最新版本号为10.2.5
BMCN OS LTS（BMCN OS长期支持版本）——该分支由后室通信委员会创建并在早期由该组织开发，后来成为M.E.G.CN官方支持分支，该分支旨在对BMCN OS的旧版本继续提供安全更新以及服务支持。目前该分支处于活跃状态且发展良好，最新版本号为1.4.2
BMCN OS community（BMCN OS社区版）——该分支由BMCN OS的社区制作而成，该分支旨在使BMCN OS变得更加开放，在后来随着BMCN Pro OS的开发而被取代，目前已经几乎停止开发，最后一个版本是4.5.3。
BMCN OS Meridian Train（BMCN OS子午列车特供版）——该分支由M.E.G.CN为子午列车特别制作，该分支仅能应用在子午列车的ZWX厢中，在子午本人的要求下，M.E.G.CN对子午列车进行了适配工作，据子午本人说该分支将会作为子午列车的备用操作系统使用。该分支在原有的BMCN OS基础上添加了特殊的GUI界面以方便操作，同时也加入了Pro分支特有的桌面环境切换器。目前该分支处于活跃状态且发展良好，最新版本号为beta1.0.2.
功能强大的系统内核
BMCN OS可以在安装过程中自主选择所需内核，BMCN OS官方版本和BMCN OS Multilingual Edition中仅支持4.5和5.0两个内核。BMCN Pro OS则支持1.0-5.5版本的所有内核。BMCN OS LTS和BMCN OS community则只支持3.5内核。
然而，在某些分支中用户可以从后联网中M.E.G.CN的社区网站上下载对应版本的内核并自行编译，但因用户操作不当而造成的设备损坏和数据丢失需用户自行负责。
长时间的安全更新与服务支持
M.E.G.CN通常会为每一个stable版本提供五年的长期支持，在这五年内用户可以收到M.E.G.CN提供的官方服务支持。M.E.G.CN提供的服务支持包括安全更新、功能更新与系统补丁。通常情况下，用户在stable的生命周期内每周都会收到三到四次系统补丁，每个季度收到一次安全更新，每年收到一次功能更新。
M.E.G.CN会为每一个testing版本提供短期支持，在这半个月内，用户可以收到stable版本的所有更新与testing版本的所有更新，每个testing版本的生命周期是半个月。
M.E.G.CN不会为unstable版本的用户提供绝大部分的支持，唯一的支持是在有新的unstable/testing版本到来时提供升级服务。
多样的生命周期
BMCN OS拥有一套生命周期系统，大部分的官方版本/分支都会遵从这一套生命周期系统，为了方便用户了解，我们制作了一张表格（仅包含常见的分支与版本）：

用户可以根据我们提供的生命周期系统来确定自己所需的系统，这对于一些拥有选择困难症的用户来说非常有效。

方便的软件管理
BMCN OS软件包格式的扩展名为brzh，brzh格式是BMCN OS系操作系统的专属软件包格式，需要配合brpt或bspp包管理器使用。以下是一些常用的包管理器命令

BRPT——Backrooms Software Package Tool
功能 具体语句
软件源配置 /pkg/brpt/brsources.list
显示官方源可用更新 brpt-bmcn update
使用官方源进行更新 brpt-bmcn upgrade
使用官方源进行版本升级 brpt-bmcn -dist upgrade
安装软件包 brpt install /xxx/xxx
从官方源安装软件 brpt-bmcn install xxx
卸载软件包 brpt remove xxx
自动检测并修复损坏的包（官方源） brpt-bmcn repair
删除无用的软件包 brpt autoremove
清理已卸载软件包的目录 brpt autoclean

BSPP——backrooms software package processor
功能 具体语句
安装brzh包 bspp -i xxx
解压brzh包 bspp -d xxx
卸载brzh包 bspp -p xxx
重新配置brzh包 bspp -r xxx
将源码打包成brzh格式的软件包 bspp -brzh /xxx/xxx
除了官方提供的包管理器之外，一些社区和组织也开发了一些联网包管理器以便于代替BMCN OS预装的老旧的应用商店。
其中最著名的便是由BMCN OS社区开发的BUSW（Backrooms Major explorer group Chinese Operating System User Software Warehouse）。该联网包管理器摒弃了在后室中效率低下的GUI界面，转而采用了高效率，高稳定性的终端界面。该包管理器支持用户在终端中搜索自己想要的软件并进行安装或下载，同时支持用户将别的分支的软件包转换成可在BMCN OS上安装的格式。附：出于安全性考虑，子午列车特供版的BMCN OS仅能使用bspp包管理器且只能安装有M.E.G.CN特发的许可证的包

如果一个软件包它的运行效率高，无许可证问题且开发者处于活跃状态，那么该软件包将有希望被选入官方源，以后它就可以通过brpt-bmcn install进行安装了。
BMCN OS安装准备
签名验证
在用户开始安装BMCN OS之前，通常需要进行验证签名，验证签名会在安装程序启动时自动完成，如果自动验证程序失败超过三次，安装程序将会切换至System terminal界面进行手动验证

check secret-key –verify
在一台已经安装过BMCN OS的终端上验证签名则需要使用以下命令

brpt install secret-key –verify

注意：如果您的安装映像不是从您的终端开发商处获取的，那么该签名则有可能是经过伪造的，我们建议您在非必要情况下不要擅自使用非官方的BMCN OS，否则将会对您的数据安全造成一定损害
另一种验证签名的方法是使用BMCN OS的开发者的签名，该方法通常用于解锁系统中的高级设置。
准备安装介质
BMCN OS可以被制作成多种类型的安装介质，例如U盘，SD卡，移动硬盘或软盘。您可以根据自身设备来选择对应的安装介质与安装映像。

选择安装环境
BMCN OS提供了两种安装方法，一种是使用较为繁琐，可配置高级功能的命令行安装界面（适合高级用户使用），另一种是使用较为简单的GUI界面，但无法配置高级功能（如磁盘分区等），在此处我们仅介绍命令行安装界面的使用方法。
安装程序验证签名结束后会自动进入命令行界面，在虚拟终端中输入gui-bmcnos install可以启动GUI安装界面，输入bmcnos-install则会进入命令行安装界面。

启动系统服务
在进入命令行安装界面后，您应当启动BMCN OS的系统服务以拉起安装过程中所需要的工具

start system-services
控制台键盘配置与字体配置
在启动系统服务之后，您就可以进行下一项工作——控制台键盘与字体配置。控制台键盘配置默认为中文键盘布局，修改布局需先列出所有可用布局，输入该命令列出键盘布局

local list-keymaps
如果您想要更改键盘布局，可以使用loadkeyboards命令，比如，要添加俄语键盘布局

loadkeyboards -rus
控制台字体位于/user/share/typeface/config目录中，直接将字体文件放入该目录中即可完成配置。

验证引导方式
要验证系统目前的引导模式，请检查BRUEFI位数

check /sys/firmware/efi/efi_size
如果命令结果为 64，则系统是以 BREFI 模式引导且使用 64 位 x64 BRUEFI。如果命令结果为 32，则系统是以 UEFI 模式引导且使用 32 位 BRUEFI，虽然其受支持，但引导加载程序只能使用 systemd-boot。如果文件不存在，则系统可能是以BIOS模式（或 CSM 模式）引导。如果系统没有以您想要的模式（BRUEFI 或 BIOS）引导启动，请您参考自己的终端说明书。

更新系统时间
使用以下命令更新系统时间

timedatectl
硬盘分区
输入该命令进入设置界面

fdisk -m
挂载硬盘分区
输入以下命令进行分区挂载，这里以root分区为例

loaddisk -/root
BMCN OS安装
选择镜像站
系统的文件 /pkg/brpt/brsources.list 中定义了软件包会从哪个镜像站下载。在 Live环境 启动的系统上，且在连接到互联网后，live会自动选择速度最快的镜像站（通常是Level 1镜像站） 我们建议将离您所处地理位置最近的镜像挪到文件的头部，同时也应该考虑一些其他标准。 要获取 mirrorlist包 的按楼层分列的原始镜像列表，在挑选了能用的镜像之后，可以执行

brpt install brpt-mirrorlist
安装系统
在您完成以上所有步骤后，安装程序会自动进行系统安装，在此过程期间，安装程序会询问您所需的工具和桌面环境，在您选择完成之后，安装程序会自动完成接下来的安装，无需您在一旁值守

结束安装
安装程序结束后会自动重启并弹出安装介质，接下来您需要自行配置您的账户和密码，在一切都配置完成之后，您就可以使用您的新系统了。

轶闻
据说，在该操作系统刚发布的时候，由于系统的不稳定，从而导致系统经常会在不经意间崩溃，因此有些流浪者埋怨“这个系统崩溃的速度比我在高危楼层死亡的速度还要快。”因而该系统被冠名“流浪者探索未半而系统崩殂”
M.E.G.CN官方的BMCN OS为付费产品。即使流浪者购买了M.E.G.CN的终端也不会附赠一个操作系统，而其他的分支却是免费的，这就导致BMCN OS本体的销售率十分惨淡。大部分流浪者都转而去下载并安装BMCN OS的多语言分支。多语言分支和原版的BMCN OS最大的区别就在于多语言分支几乎支持所有人类仍然在使用的语言（可以理解为免费的BMCN OS原版）。后来为了增加更多的使用用户，M.E.G.CN不得不宣布BMCN OS限时免费。
BMCN OS可能是后室中使用频率最高的操作系统。
