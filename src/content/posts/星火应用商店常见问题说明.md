在很多情况下，对于一些刚入门的debian系Linux用户来说安装软件可能并不是一件容易的事，尽管部分发行版自带图形化软件安装器。但众所周知，这些软件安装器多少没有终端命令行好用。

而在一些情况下，由于deb包的依赖问题以及一些其他特殊情况，会导致星火商店安装失败。基于一些用户反馈的问题，在此写成一篇文章以供参考。

# 安装星火应用商店
## debian系Linux
读前须知：该部分教程仅适用于debian系Linux，在其他发行版安装星火商店请见后文。

### **第一部分——准备工作**
在开始安装之前，我们需要先进行一些准备工作。
#### 1.进行系统更新
在您开始安装之前，请务必将您的操作系统保持在一个最新状态。执行以下命令进行更新
``` bash
sudo apt update  #更新软件源
sudo apt upgrade  #开始更新
```

#### 2.下载软件本体及依赖包
在星火应用商店的[gitee](https://gitee.com/spark-store-project/spark-store/releases/tag/4.2.12)页面中下载deb包以及准备所需的[依赖](https://spark-app.store/download_latest)。<br>
PS：在下载之前请确认您所下载的应用架构是否和您的计算机CPU架构相同，使用以下命令来确认您的计算机CPU架构：
``` bash
uname -m
```
请根据您的终端输出内容来选择对应架构的软件包。

### 在准备好软件本体以及依赖包后进入**第二部分——开始安装**
当您完成准备工作之后便可以开始安装软件包。执行以下命令安装软件包
``` bash
cd /home/(你的用户名)/download  #在大多数情况下，从浏览器下载的文件默认存储在该文件夹，切换到该目录以准备安装
sudo apt-get install (文件名)  #请将文件名以及括号删除，并替换为您所下载的软件包名称（别忘了把.deb后缀也复制上）
```
如果您的发行版是主流发行版且版本较新，那么您可以直接执行安装命令而无需进行其他操作；如果您的发行版是非主流版本（如麒麟Linux等其他非主流发行版）且版本较旧（如debian10，ubuntu20.04等）那么您就需要在开始安装之前先手动安装星火提供的依赖包或自行编译依赖（或许考虑去整个ACE也不错？）。

存储依赖包的文件夹中会有一个脚本，执行该脚本以开始进行依赖安装，或者您也可以在输入apt-get install 后将所需安装的依赖全部（一次性选中）拖动到终端窗口中，这同样可以进行依赖安装。

在成功安装依赖包之后您就可以回到前文进行软件包本体安装了。如果不出意外的话您可能会在终端看到一行报错，大致内容如下
``` bash
N: 由于文件'/home/xxx/xxxx.deb'
无法被用户'xxx'访问，
已脱离沙盒并提权为根用户来进行下载。
- pkgAcquire::Run (13: 权限不够)
```
该报错并不影响星火商店以及所安装的软件的正常使用，系正常现象。

## arch系Linux
在arch系Linux安装星火应用商店需要安装ACE兼容环境，执行以下命令以开始安装
``` bash
yay -S amber-ce-bookworm
```
如您未安装yay可以尝试执行以下命令
``` bash
sudo pacman -S yay
```
或参考[arch wiki](https://wiki.archlinuxcn.org/wiki/Yay)页面自行编译。<br>
在安装完成ACE兼容环境后请参考debian系linux安装教程进行安装

## RedHat系Linux
**该部分摘自ACE gitee文档**
### 获取编译依赖
``` bash
sudo dnf install rpm-build debootstrap dpkg which systemd-container jq -y
```
### 下载
下载并解压到用户home目录<br>

#### 获取最新版本号
``` bash
./rpmbuild/SPECS/get-ver.sh
```
#### 下载最新源码包
``` bash
cd rpmbuild/SOURCES && ./get-tgz.sh && cd -
```
### 构建
``` bash
cd rpmbuild && rpmbuild -ba SPECS/bookworm-compatibility-mode.spec
```
### 安装
``` bash
cd rpmbuild/RPMS/*/ && sudo dnf install *.rpm -y
```
在安装完成ACE兼容环境后请参考debian系linux安装教程进行安装

# 关于使用星火商店时遇到的一些小问题
Q：为什么有时我下载软件它会一直卡在0kb/s <br>
A：这是由于星火的服务器压力较大而导致的问题，目前似乎已被解决。

Q：我是debian/(其他发行版)用户，为什么商店里所有的软件都显示无法兼容该发行版。 <br>
A：该原因主要是因为星火没有制作其他部分debian系Linux发行版的软件兼容提示，目前商店的软件兼容提示仅有ubuntu，deepin，UOS等。不影响使用，兼容问题请以软件简介和实际安装为准。

Q：我想在非debian系Linux上安装星火应该怎么做 <br>
A：通常情况下，我们推荐您使用ACE兼容环境来进行安装，该项目的地址：https://gitee.com/amber-ce/amber-ce-bookworm <br>
如果您无法独自解决该问题，或者有其他需求，您也可以加入星火商店用户交流群寻求一些其他的可能是有偿的支持（毕竟星火的主线支持还是debian系Linux，开发这些兼容性软件不易，请理解）
